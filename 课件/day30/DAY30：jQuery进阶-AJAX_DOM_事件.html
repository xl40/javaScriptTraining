元素节点操作：
    1.创建节点。
        var box=$('<div id="box">节点</div>');    //创建一个节点
        $('body').append(box);    //将节点插入到<body>元素内部

    2.插入节点
        append(content) 向指定元素内部后面插入节点 content
        appendTo(content) 将指定元素移入到指定元素 content 内部后面
        prepend(content) 向指定元素 content 内部的前面插入节点
        prependTo(content) 将指定元素移入到指定元素 content 内部前面
        after(content) 向指定元素的外部后面插入节点 content
        before(content) 向指定元素的外部前面插入节点 content

    3.节点复制
        $('body').append($('div').clone(true));     //复制一个节点添加到 HTML 中
        注：clone(true) 参数为空时，只复制元素和内容，不复制事件。为 true 时，也拷贝事件。

    4.节点的删除
        $('div').remove();    //直接删除 div 元素
            remove()方法可以接收一个参数，过滤需要删除的元素。
        $('div').empty();     //删除 div 所有的子元素

    5.节点的替换
        $('div').replaceWith('<span>节点</span>');     //将 div 替换成 span 元素

二、jQuery事件
1. 常规事件绑定
    jquery的事件就是将JavaScript事件简写（省略了on）,然后在后面加上括号。(注：前缀必须是jQuery对象)

    原生绑定方式：
        document.onclick = function(){}
    Jq绑定方式：
        $(document).click(function(){})
    

2. on绑定，off删除
    on方法绑定事件
        jqDOM元素.on("click", [selector], [data], function(){})

    off方法删除事件
    on绑定的事件存在事件命名空间，以供off删除时找到指定事件

        jqDOM元素.on("click.a", function(){})
        jqDOM元素.on("click.b", function(){})
        jqDOM元素.off("click.a")

    on方法直接支持事件委托
        jqDOM元素.on("click", "要委托的子元素选择器", function(){})

    多个事件，共用一个事件处理函数
        $('input').on('mouseover mouseout',function(){})

    多个事件，使用不同事件处理函数
        $('input').on({
            mouseout:function(){},
            mouseover:function(){}
        })

    一个事件，多个不同的事件处理函数
        jqDOM元素.on("click", function(){})
        jqDOM元素.on("click", function(){})
        
3. one绑定
    绑定的事件只能被触发一次，然后会被删除
        jqDOM元素.one("click", function(){})

4. hover绑定
    为 鼠标进入mouseenter 和 鼠标离开mouseleave 事件的封装
    
        jqDOM元素.hover(function(){}, function(){})
    
    第一个参数为鼠标进入的事件处理函数
    第二个参数为鼠标离开的事件处理函数

5. bind绑定，unbind删除（新版本中已被抛弃）
    新版中已被废弃，全面被on/off方法取代

6. trigger和triggerHandler
    模拟事件触发，自定执行指定事件
    
        jqDOM元素.trigger("click”);

    trigger默认触发事件冒泡
    triggerHandler不会触发事件冒泡

7. 事件对象
    JavaScript事件对象是浏览器默认传入的，但是对于浏览器的兼容问题，我们需要对事件对象进行兼容。很烦！但是jQuery已经帮我们解决了所有兼容性的烦恼，并且给我们添加了很多有用的方法。

        event.target              获取绑定事件的DOM元素
        event.type                获取事件的类型
        event.data                获取事件中传递的数据
        event.pageX/pageY         获取根据页面原点的X,Y值
        event.screenX/screenY     获取根据显示器窗口的X,Y值
        event.offsetX/offsetY     获取根据父元素X,Y值
        event.which               获取当前鼠标点击的键1,2,3
        event.altKey/shiftKey/ctrlKey/   返回 true、false
        event.preventDefault()    阻止默认行为
        event.stopPropagation()   取消事件冒泡

        return false              阻止默认事件和事件冒泡，双层阻止，小心使用


三、Ajax
异步和同步

    使用 Ajax 最关键的地方，就是实现异步请求、接受响应及执行回调。那么异步与同步 有什么区别呢？我们普通的 Web 程序开发基本都是同步的，意为执行一段程序才能执行下 一段，类似电话中的通话，一个电话接完才能接听下个电话；而异步可以同时执行多条任务， 感觉有多条线路，类似于短信，不会因为看一条短信而停止接受另一条短信。Ajax 也可以 使用同步模式执行，但同步的模式属于阻塞模式，这样会导致多条线路执行时又必须一条一 条执行，会让 Web 页面出现假死状态，所以，一般 Ajax 大部分采用异步模式。

 
 

    jQuery对Ajax采用了三层封装，其中最底层的是$.ajax(),中层封装是$().load()，$.get()，$.post，最顶层是getScript和getJSON()。

$().load()方法
    $().load()方法是局部方法
        $().load(url, data, cb)

    1.url: 要请求的地址，可以提供筛选功能。
        $().load('data/test.html ')
        $().load('data/text.html .box')>>>筛选功能

    2.data: 发送的数据，可选
        $().load('data/test.html',{ data:'admin' })

    3.cb: 回调函数function(response,status,xhr){}
        $().load('data/test.html',{ data:'admin' },function(response,status,xhr){})


    response：返回结果。

    status ：状态字符，success，或者error等

    xhr ：XMLHttpRequest对象
 
xhr有四个常用属性。
属性名
说明
responseText
作为响应主体被返回的文本
responseXML
如果响应主体内容类型是"text/xml"或"application/xml"， 则返回包含响应数据的 XMLDOM 文档
status
响应的 HTTP 状态（状态码）
statusText
HTTP 状态的说明



$.get()和$.post()
    $.get()和$.post()方法是全局方法，使用语法一致，此处以$.get()作为说明
        $.get(url, data, cb, dataType)
        
    url: 地址,必填
    data: 要发送的数据，可选，
    cb: 请求成功的回调函数
    dataType: 强制转换接收到的数据类型
        常用的数据格式有html，text，json。
        如果写入数据格式，那么默认会强制转换为该格式
        如果传入错误格式，那么不会执行请求成功的回调函数
    
    注：一般情况下，dataType都是智能判断，并不需要人为设置，也建议不要使用这个参数。
    
    $.get()和$.post()的差别:$.get()是以get方式提交，$.post()方式是以post方式提交。

        1.get方式提交数据是吧数据放在浏览器网址上面的，post是通过http消息，实体提交的。
        2.get提交方式有大小限制，限制在2KB，而post方式不收大小限制。
        3.get方式因为在浏览器中会被历史记录缓存，所以这种提交数据方式并不安全，post方式没有这种问题。
        4.在服务器端（php语言），get方式通过$GET[]方式获取，post通过$POST[]方式获取。

$.ajax()方法
jQuery封装的ajax的底层方法，可以实现以上方法所有功能，可以直接使用一个对象参数，配置所有信息

    $.ajax({
        url:'',         // 请求地址
        type:'',        // 用什么方式提交
        data:{},        // 传入数据用键值对方式传入
        success:function(){},    // 请求成功时的回调函数
        error:function(){},      // 请求失败时的回调函数
        beforeSend:function(){}, // 执行send之前的回调函数
        timeout:1000,            // 请求超时的时间
        global:true,             // 是否触发ajax的全局事件
    })



jQuery的全局事件（生命周期）
全局函数
 何时被触发
参数
ajaxStart
在jQuery AJAX函数或命令发起时，但在XHR实例被创建之前
类型被设置为ajaxStart的全局回调信息对象
ajaxSend
在XHR实例被创建之后，但在XHR实例被发送给服务器之前
类型被设置为ajaxSend的全局回调信息对象；XHR实例；$.ajax()函数使用的属性
ajaxSuccess
在请求已从服务器返回之后，并且响应包含成功状态码
类型被设置为ajaxSuccess的全局回调信息对象；XHR实例；$.ajax()函数使用的属性
ajaxError
在请求已从服务器返回之后，并且响应包含失败状态码
类型被设置为ajaxError的全局回调信息对象；XHR实例；$.ajax()函数使用的属性；被XHR实例返回的异常对象(如果有的话)
ajaxComplete
在请求已从服务器返回之后，并且在任何已声名的ajaxSuccess或ajaxError回调函数已被调用之后
类型被设置为ajaxComplete的全局回调信息对象；XHR实例；$.ajax()函数使用的属性
ajaxStop
在所有其他AJAX处理完成以及任何其他适用的全局回调函数已被调用之后
类型被设置为ajaxStop的全局回调信息对象
